# Guru Maps Integration

🇪🇸 [Versión en español](gurumaps.es.md)

NookMesh’s primary visualization integration is currently designed for **Guru Maps**.

Guru Maps can consume custom GeoJSON sources and apply advanced visual styling through MapCSS, making it a particularly powerful client for real-time location visualization.

---

## What this integration provides

With Guru Maps, NookMesh can display:

- real-time positions
- short identifiers (`tid`)
- dynamic visual freshness indicators
- movement direction (when available)
- battery level
- speed
- GPS accuracy
- altitude
- enriched contextual information

Depending on the authenticated user’s role, some sensitive metadata may be automatically hidden.

For example, non-`staff` users do not receive:

- source device
- connection type
- internal groups

Privacy logic is enforced in the backend before data delivery.

---

## General flow

```text
OwnTracks
   ↓
MQTT
   ↓
Recorder
   ↓
GeoJSON Worker
   ↓
Protected API
   ↓
Guru Maps
```

Guru Maps consumes already authenticated, filtered, and enriched GeoJSON.

All sensitive logic happens in the backend.

---

## Integration format

NookMesh exposes data via:

```text
GeoJSON
```

Typical endpoint:

```text
https://geojson.yourdomain.com/nookmesh.geojson?token=YOUR_TOKEN
```

Guru Maps consumes this endpoint as a custom GeoJSON source.

---

## Authentication

Access is protected through an individual API token.

Format:

```text
?token=YOUR_TOKEN
```

This enables:

- individual access control
- selective revocation
- personalized visibility
- user isolation

Guru Maps does not participate in internal authentication.

It only consumes the authorized result.

## Quick import template

NookMesh includes a ready-to-import Guru Maps overlay template:

```text
docs/assets/gurumaps/nookmesh_gurumaps_overlay.ms
```

Contents:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<customMapSource overlay="true">
    <name>🛰️ NookMesh</name>
    <minZoom>0</minZoom>
    <maxZoom>22</maxZoom>
    <geojson url="https://your_host/nookmesh.geojson?token=your_token" updateInterval="0.05"/>
    <style url="https://your_host/nookmesh_v1.mapcss?token=your_token"/>
</customMapSource>
```

You only need to replace:

- `your_host`
- `your_token`

according to your deployment.

## Current visual model

The current integration uses enriched rendering generated by the API.

For each visible entity, NookMesh generates **two coordinated GeoJSON features**:

### Icon feature

Represents the main icon.

Example:

```json
"render": "icon"
```

Includes:

- dynamic SVG
- freshness-based color
- heading marker (if available)
- contextual description

---

### Label feature

Represents the text label.

Example:

```json
"render": "label"
```

Includes:

- `tid`
- name
- contextual description

---

This allows separation of:

- iconography
- text
- visual behavior

through MapCSS.

---

## Current visual indicators

The current system uses:

```text
< 5 min      → green
5–29 min     → orange
30–60 min    → red
> MAX_AGE    → hidden
```

Additionally:

- `staff` users use differentiated internal styling
- other users use alternative styling
- heading generates an oriented black marker

## Viewer self-exclusion

NookMesh supports:

```env
EXCLUDE_VIEWER_IN_OUTPUT=true
```

When enabled:

the authenticated user **does not receive their own location** inside the GeoJSON.

This avoids duplicate markers when Guru Maps already displays the device’s local GPS position.

Strongly recommended.

## Guru Maps cache

Guru Maps may cache results.

This can cause symptoms such as:

- delayed updates
- seemingly frozen positions
- styles appearing not to refresh

In many cases, this is not a backend issue.

---

- access control

## Troubleshooting

### Nothing appears

Check:

- correct token
- correct endpoint
- accessible API
- valid HTTPS
- API container logs

---

### OwnTracks publishes but Guru Maps shows nothing

Check:

- recorder
- worker
- `nookmesh.geojson` generation
- visibility permissions
- data freshness (`MAX_EDAD_MIN`)

---

### I only see some users

Check:

- shared groups
- `oculto_para`
- authenticated user token
- operational filters

---

### I see data but no styling

Check:

- MapCSS URL
- HTTP/HTTPS accessibility
- style endpoint token
- style compatibility

---

### My location appears duplicated

Check:

```env
EXCLUDE_VIEWER_IN_OUTPUT=true
```

---

### Changes take too long to appear

Possible causes:

- Guru Maps cache
- client refresh behavior
- worker export interval
